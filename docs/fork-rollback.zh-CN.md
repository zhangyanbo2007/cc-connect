# 会话分叉与回撤

> [English](./fork-rollback.md)

cc-connect 支持会话分叉和回撤，适用于实现了 `SessionForker` 接口的 agent（当前仅 Claude Code）。

---

## /fork — 分叉会话

从当前会话的某个轮次创建一个新的分支会话。原会话保持不变，分叉会话继承到选定轮次的对话历史。

### 用法

| 命令 | 说明 |
|------|------|
| `/fork` | 弹出卡片选择最近 10 轮作为分叉点 |
| `/fork [名称]` | 分叉整个会话，指定自定义名称 |
| `/fork [名称] N` | 从倒数第 N 轮分叉，指定名称 |

不指定名称时自动命名：**`fork 1 of <当前名称>`**、**`fork 2 of <当前名称>`**……

### 卡片选择（飞书、Slack、Telegram）

在支持交互卡片的平台上，输入 `/fork`（不带参数）会原地弹出卡片，列出最近 10 轮对话摘要，每轮显示用户消息预览。选择一轮或"整个会话"即可从该点分叉。

```
┌─ 从哪一轮分叉？ ────────────────────────┐
│                                         │
│  🔽 选择轮次:                            │
│    倒数第1轮: 南沙这边想做个副业...        │
│    倒数第2轮: 有什么风险吗                │
│    倒数第3轮: 代办不是还要跑腿哦啊         │
│    ...                                  │
│    整个会话                              │
│                                         │
└─────────────────────────────────────────┘
```

选中后卡片原地更新显示结果：

```
┌─ 从哪一轮分叉？ ────────────────────────┐
│                                         │
│  ✅ 分叉成功: fork 1 of 意识探讨 (s8)    │
│                                         │
│  [← 返回]                               │
└─────────────────────────────────────────┘
```

### 示例

```
用户:  /fork
Bot:   弹出轮次选择卡片 → 用户选择"倒数第3轮"
Bot:   ✅ 分叉成功: fork 1 of 南沙副业 (s9)，从第3轮开始
```

直接指定参数：

```
用户:  /fork 意识探索 2
Bot:   ✅ 分叉成功: 意识探索 (s10)，从第2轮开始
```

## /rollback — 回撤轮次

从**当前**会话中移除最近 N 轮对话。截断 agent 的 JSONL 历史，模型只看到剩余上下文。

### 用法

| 命令 | 说明 |
|------|------|
| `/rollback` | 弹出卡片选择回撤点 |
| `/rollback N` | 直接回撤最近 N 轮 |

### 卡片选择

在支持的平台上，`/rollback`（不带参数）弹出卡片。选中后卡片更新，显示回撤了多少轮、还剩多少轮。

```
┌─ 回撤 ─────────────────────────────────┐
│                                         │
│  ✅ 已回撤 3 轮，剩余 4 轮              │
│                                         │
│  [← 返回]                               │
└─────────────────────────────────────────┘
```

### 示例

```
用户:  /rollback 2
Bot:   ✅ 已回撤 2 轮，剩余 8 轮
```

回撤后，agent 的上下文窗口只包含截断后的历史。后续消息从该点继续。

---

## 关键区别

| | `/fork` | `/rollback` |
|---|---|---|
| 创建新会话？ | 是（分支会话） | 否（修改当前） |
| 原会话？ | 保持不变 | 被截断 |
| 可以切换？ | 是，通过 `/switch` | — |
| 自动命名？ | `fork N of <名称>` | — |

---

## 会话名称保护

分叉和回撤都会保护当前会话名称在 agent 的 JSONL 文件中。回撤截断可能从文件末尾删掉最新的 `custom-title` 条目——cc-connect 在截断后自动重写标题，确保 VSCode / CLI 始终显示正确的名称。
